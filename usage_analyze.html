mmanager analyze

Using:  mmanager analyze 
$ bin/mmanager analyze -h
Usage: mmanager analyze

Common options:
    -V, --version                    Show the version number
    -h, --help                       Display ze help
Before running analyze, you should run import to have at least one path in your media library otherwise analyze will have nothing to process.  Running analyze will go through your media library and for every file that has not been positively identified it will collect search terms from the filename, and consult the configured scrapers to look for possible results.  It will then attempt to match each result returned to the filename looking for the best match.
Currently, it will only positively identify on an exact match - which is not very often.
The intent is to add both result-scoring code to assess the best possible match, and also to add an interactive mode which will prompt the user to choose the best result if no one particular match can be singled out automatically.
The positively identified matches are stored in a file so that you only have to process each file once.  This information is used to categorize your media to add to your Library directory with the remap command.Examp
